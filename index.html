About Project - It's Urgent

  • Mentor: Akshat Tripathi
  • Description: Create a Flutter app that helps users priorotize their task, and make sure no important notification is missed.

What I will do - Abstract

During the course of this project I will develop a Flutter application with Firebase as backend that enable users to decide weather or not they want to be disturbed, while still not missing very important communications.

The app will work in a messaging app's fashion except that communications cannot be carried out on the app like a messaging platform.

  1. Users sync contacts
  2. Set their current status from avaiable busy and dnd
  3. Set a challenge that must be completed inorder to send notification
  4. Select a user from their contact list to call
  5. Fulfill their challenge (if any)
  6. Convey the very important message

Apart from that, I would like to implement the following ideas:

  • A feature to schedule meeting for sometime in future with a user
  • Automatically set user status based on location and/or time

How - Detailed Approach

Week 1

In the first week I will plan and discuss the app's core features with mentors. I will also create a basic UI mockup and plan a database architecture which suits best for this usecase. My database of choise will be Firebase Firestore as it offers seemless integration with any flutter project and is fairly scaleable with a generous free tier.

Week 2

For week 2 I will be setting up firebase cloud messaging for sending notifications to users whenever someone needs to contact them. I will add 3 ways of Authentication for the app, Google Auth, Email and Password, and Phone number.

All three of these will be stored in a different collection called Auth with structure that's similar to this illustration I made:

Here Auth and Data are two database collections. Auth has 3 sub-collections, each storing authentication data. A key, mapping to a usrId. On the other hand, the Data collection has documents with usrId as their IDs.
After authenticating, we can access a user's data using their secret ID. This also keeps their data private and secure.

Week 3

In Week 3 I will implement a 'Find friends' feature where user can sync their contacts and we will check who in their contact list is using the app. We can then store this list in local storage to maintain privacy and reduce storage on our firestore.

I will implement a system like of WhatsApp's. It shows recent conversation on opening app. These will be the contacts in our local storage. With a seperate FAB to view all other, at which point we will go through the entire contact list to check for users on the app.

Also I will add a way to set status from avaialble, busy and dnd and store it in firestore.

Week 4

For week 4, I have planned to implementing the setting a challenge feature. This will be avaialble from settings of app, and can be a password, passphrase or a math operation etc. Along with that, I will add feature to send notification to users to contact them using Cloud Messaging.

Week 5

Now user will be able to set the priority of their Notification from low, medium to high. Along with that I will implement a way to solve challenges, now user will have to solve challenge before contacting if the user is in DND mode.

Week 6

Week 6 won't be seeing a lot of new progress, I will self evaluate the app as well as submit it for evaluation by mentors and work upon their feedback. By this point the app will have a basic version ready which has been planned orignally.

Week 7

In Week 7, I will be adding a feature to send a meeting request for sometime in future. The other user will be able to accept or reject this request by the end of this week.

Week 8

I will now create a cron job, that will check for elegible scheduled meetings. If any exist, notifications will be sent out to both people using Firebase Cloud Messaging.

Week 9

In this week I will perform self evaluation, ask mentors for their feedback and suggestions. I will also refactor the code as necessary and write the documentation, user manual and developer guide in this week.

Week 10

For week 10 I will be adding a feature insipired by Ultimate Alarm Clock. I will implement a feature to automatically set status based on location and time. We will need user's GPS access for this.

Week 11

Week 11 will be dedicated to improvement and testing. I will also complete any remaining tasks from previous weeks. I will also be working upon documentation and guide in this week. The UI will be refined, with appropriate graphics, the UX will be improved and security tested.

Week 12

In Week 12, the final week for the project, I will be wrapping up the project with feedback from mentors and improving upong them. Finalizing the documentation and releasing the app on Google Play Store
